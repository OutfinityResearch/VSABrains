Experiment 6: Literature & Dialogue Semantics

Goal: test whether semantic frame indexing makes complex, high‑level questions faster than a naive scan. This experiment does not require an LLM; it simulates a semantic encoder and focuses on query speed.

Plain-language version: if we pre‑count semantic frames (emotion, theme, tone), we should answer “what’s dominant?” much faster than scanning the whole story every time.

How this experiment works

  • Generate a large stream of semantic facts (frame, subject, value).
  • Build an index: for each frame, count the most frequent values.
  • Run the same queries in two ways:
  1. Naive scan — count values by scanning all facts.
  2. Indexed — answer directly from the pre‑built frame index.

What this experiment proves

  • Frame indexing makes high‑level semantic queries much faster.
  • The speedup grows with the number of facts.
  • This validates the design idea behind “frames” and “CNL profiles”.

Output fields

Field Meaning (plain language)
factsHow many semantic facts were generated.
queriesHow many frame queries were timed.
naiveMsTotal time for naive scan queries (milliseconds).
indexedMsTotal time for indexed queries (milliseconds).
speedupNaive time ÷ indexed time (higher is better).
