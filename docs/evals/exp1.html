Experiment 1: Reference-Frame Alignment

Goal: show that multiple columns with different offsets can still agree on the same latent situation. The key signal is that consensus should outperform any single column under ambiguity and noise.

Plain-language version: several small “witnesses” see the same story with different noise. We want the group vote to be more reliable than any single witness.

Alignment means that even if columns start in different places, they can still agree on which location best matches the query window.

How this experiment works

  • Generate a synthetic story with known locations.
  • Give each column a slightly different noisy view of the same window.
  • Ask “where is the window?” and compare single-column vs consensus.

How to read the metrics

  • Loc@1 = top-1 accuracy (best guess is correct).
  • Loc@5 = top-5 accuracy (correct location is in the top 5).
  • Consensus Gain = how much better the vote is vs one column.

Why this matters

If alignment fails, the architecture is unreliable under noise. Passing here is the minimum proof that “many small columns” is not just extra compute—it is extra robustness.
